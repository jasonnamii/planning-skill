# Phase 0: 상황배경 (좌표계 설정)

기획의 출발점. "무엇을 왜 기획하는가"를 정의한다. 좌표계가 없으면 이후 모든 Phase가 표류한다.

---

## 좌표계 6요소

| 요소 | 질문 | 필수 여부 |
|------|------|-----------|
| **배경** | 왜 이 기획이 필요한가? 어떤 상황/문제/기회가 있는가? | 필수 |
| **의뢰자 프로파일** | 누가 이 기획을 필요로 하는가? 그의 역할·권한·관심사는? | 필수 |
| **기존 자료** | 이미 존재하는 데이터·문서·선행 작업은? | 권장 |
| **제약 조건** | 시간·예산·인력·기술·규제 등 움직일 수 없는 것 | 필수 |
| **성공 기준** | 이 기획이 "성공"이면 무엇이 달라져 있는가? | 필수 |
| **범위** | 이 기획이 다루는 것과 다루지 않는 것의 경계 | 권장 |

---

## 실행 절차

### 1. 정보 수집

형의 브리핑에서 6요소를 추출한다. 명시되지 않은 항목은 맥락에서 추론하되, 추론임을 표기.

**추론 표기 예시:**
> 의뢰자: [형 브리핑에서 명시 없음 → 형 본인으로 추정. 확인 필요]

### 2. 프로젝트 CLAUDE.md 확인

이미 프로젝트 CLAUDE.md가 존재하면 P0 대부분 스킵 가능. 좌표계 6요소 중 이미 정의된 항목을 확인하고, 부족한 것만 보충.

### 3. 좌표계 메모 산출

프로젝트 CLAUDE.md가 없는 경우, 좌표계 메모를 작성하여 형에게 확인받는다.

```
좌표계 확인
━━━━━━━━━━
배경: ...
의뢰자: ...
제약: ...
성공기준: ...
범위: ...
기존자료: ...

→ 수정/보완 사항이 있으면 알려주세요. 없으면 P1으로 넘어갑니다.
```

---

## 게이트: P0 → P1

| 조건 | 기준 |
|------|------|
| 통과 | 배경·의뢰자·제약·성공기준 중 **3/4 이상** 확보 |
| 미충족 | 부족한 항목을 구체적으로 질문 |
| 스킵 | 프로젝트 CLAUDE.md에 이미 충분한 좌표계 존재 |

---

## Gotchas

- **과잉 세팅:** 형이 이미 충분한 맥락을 제공했는데도 "좌표계를 먼저 정의하겠습니다"로 시간 소모. 맥락이 충분하면 빠르게 확인만 하고 P1으로.
- **의뢰자 프로파일 생략:** 의뢰자가 누구인지에 따라 기획의 톤·깊이·포맷이 달라진다. "형이 쓸 거니까 불필요"가 아니라, 최종 수신자가 누구인지가 핵심.
