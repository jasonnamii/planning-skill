# Phase 1: 가설적 목표

"완전한 X"를 정의하고, 현재 상태와의 갭을 구조적으로 드러낸다. 갭이 이후 리서치의 축이 되고, 기획안의 뼈대가 된다.

---

## 핵심 원리: 갭 분석이 구조를 만든다

기획의 구조는 "있어야 할 것 - 있는 것 = 없는 것"에서 나온다. 이 갭 목록이 곧 리서치 축이 되고, 기획안의 섹션이 된다. 갭 없이 구조를 먼저 잡으면 "있을 법한 항목 나열"이 되어 스파인이 사라진다.

---

## 실행 절차

### 1. "완전한 X" 정의 (절대자 트리거)

trigger-dictionary 스킬에서 **절대자** 트리거를 호출한다.

절대자가 하는 일:
- 형의 프레임(P0 좌표계) 기반으로 "X가 완전하려면 무엇이 갖춰져야 하는가"를 MECE 분해
- 빠진 항목을 자율적으로 탐색하여 추가
- 프레임 자체의 변경은 하지 않음 (맹점보완으로 플래그만)

### 2. 갭 매트릭스 작성

절대자 결과를 기반으로 현재 상태를 대조한다.

| 달성 조건 | 현재 상태 | 갭 수준 | 비고 |
|-----------|-----------|---------|------|
| 조건 A | 이미 충족 | 🟢 | — |
| 조건 B | 부분 충족 | 🟡 | 어떤 부분이 부족한지 |
| 조건 C | 미충족 | 🔴 | 왜 없는지, 확보 방법 가설 |
| 조건 D | 미충족 | 🔴 | — |

### 3. 가설적 목표 수립

갭 매트릭스의 🔴 항목들을 해결한 상태가 "가설적 목표". 이 시점에서는 검증 전이므로 "가설"이다.

```
가설적 목표: [🔴 갭이 모두 해소된 상태를 1문장으로]
핵심 🔴 갭: N개
리서치 필요 축: [🔴 갭에서 도출된 조사 방향]
```

---

## 게이트: P1 → P2

| 조건 | 기준 |
|------|------|
| 통과 | 갭 매트릭스 완성 + 🔴 항목 1개 이상 |
| 미충족 (갭 없음) | 🔴가 0개면 "기획 불필요 — 실행 단계로 이동" 보고 |
| 미충족 (분해 불충분) | MECE 누락 의심 시 절대자 재실행 제안 |

---

## Gotchas

- **갭 매트릭스 없이 리서치 시작:** "일단 조사부터" → 방향 없는 리서치 → 시간 낭비. 갭이 리서치의 질문을 만든다.
- **🟡를 🟢로 처리:** 부분 충족을 충족으로 넘기면 기획에서 해당 영역이 빠진다. 🟡는 "어떤 부분이 부족한지"를 명시해야 한다.
- **절대자 확장 과잉:** MECE를 위해 달성 조건을 무한 확장. 범위(P0)를 벗어나지 않는 선에서 분해하고, 범위 밖 항목은 맹점보완으로 플래그.
