# Phase 5: 기획안 진화

P3 뼈대(v1) + P4 점프 결과를 통합하여 기획안을 반복 진화시킨다. v1→v2→v3+.

---

## 핵심 원리

기획안은 한 번에 완성되지 않는다. 뼈대(v1) → 살(v2) → 피부(v3+)의 순서로 밀도를 높이며, 각 단계마다 형의 피드백을 반영한다.

---

## 진화 단계

| 버전 | 별칭 | 내용 | 형의 역할 |
|------|------|------|-----------|
| **v1** | 뼈대 | P3 스켈레톤 + P4 점프 통합. 스파인·대방향·핵심액션 제목 | 구조 승인/수정 |
| **v2** | 살 | 각 섹션에 근거·수치·논리 채움. SHE 패턴 적용 | 내용 검토/보완 지시 |
| **v3+** | 피부 | 포맷 최종화, 기획언어 정제, QC | 최종 승인 |

### v1 → v2 전환

v1의 각 섹션을 **SHE 트리거**(trigger-dictionary)로 풍성하게 만든다.

SHE = Setup(상황설정) → Hook(긴장·질문) → Evidence(근거·해결)

각 섹션이 "왜 이 방향인가"를 SHE 흐름으로 설명할 수 있어야 한다.

### v2 → v3+ 전환

내용이 확정되면 형이 지정한 포맷으로 최종화한다.

---

## 포맷 적용

**기본:** Obsidian .md + UP §C 모래시계 구조 + 청킹 디자인

**형 지정 시:** 형이 직접 지정한 스타일 체계를 적용. 예시:

```
형 지정 예: "청킹 디자인 + 40줄 + 부록접기 + HTML div 1번"
→ UP §C.CHUNK_DESIGN 적용
→ 블록당 ~40줄 가이드
→ 부록 섹션은 Obsidian collapse(접기)
→ html-div-style 스킬 호출 (Style 1 Corporate Minimal)
```

형이 지정하지 않은 항목은 UP §C 기본값.

---

## QC 절차

v3+ 단계에서 기획안 QC를 실행한다. UP §C.DELIVERABLE_3STEP의 ③QC 기준 적용.

### 기획언어 6대 금지패턴

기획 산출물에서 반드시 제거해야 할 패턴. (출처: 기획언어_금지패턴_UP용.md)

| # | 금지 패턴 | 예시 (✗) | 대체 (✓) |
|---|-----------|----------|----------|
| 1 | 번호 단계 (Phase/Layer/Step) | "Phase 1: 시장분석" | "시장이 두 배로 커진다" (내용 중심 제목) |
| 2 | 이니셜·약어 | "CX 개선", "UX/UI" | "고객 경험 개선", "사용자 화면·경험" |
| 3 | 방법론 프레임워크 이름 | "Blue Ocean 전략 적용" | 전략 내용을 직접 서술 |
| 4 | 빈 상위개념 | "핵심 전략 3가지" | 전략을 각각 직접 명명 |
| 5 | 난해 전문용어 | "해자 구축", "Cold Start" | "방어벽 구축", "초기 가치 전략" |
| 6 | 표준 용어 재정의 | "MVP를 재정의하면..." | 기존 정의 그대로 사용 |

### QC 체크리스트

| 순서 | 항목 | 기준 |
|------|------|------|
| ① | 수치 정합성 | UP §D.NUM6 — 금액·지표·비율 교차 검증 |
| ② | 논리 완결성 | UP §D.V3L — 인과·전제·일관성 |
| ③ | 톤·어휘 | 기획언어 6대 금지 + UP §C.TONE + INTERNAL_VOCAB |
| ④ | 청킹 디자인 | UP §C.CHUNK_DESIGN 준수 |
| ⑤ | 스킬 발동 | project-guideline-updater (프로젝트폴더 해당시) |

### 제출청소 (선택)

외부 수신자에게 전달되는 기획안인 경우, **제출청소 트리거**(trigger-dictionary)를 호출하여 내부 언어·작업 흔적·AI 흔적을 제거한다.

### 이쁘니 (선택)

형이 시각적 정돈을 요청하면 **이쁘니 트리거**를 호출하여 최종 시각 정리.

## Gotchas

- **v1에서 바로 포맷 작업:** 뼈대 단계에서 HTML div나 시각 정리를 하면, 구조 변경 시 재작업. 내용 확정(v2) 후 포맷(v3).
- **기획언어 금지패턴 무시:** "Phase 1", "핵심 전략 3가지" 같은 패턴이 잔존하면 기획의 설득력 저하. v3 QC에서 반드시 점검.
- **SHE 과잉 적용:** 모든 문단에 SHE를 강제하면 문서가 장황. 핵심 섹션(진단, 전략 방향)에 집중 적용.
- **형 미지정 포맷을 자의적 결정:** "HTML div가 좋을 것 같아서 적용했습니다" → 금지. 형이 지정한 것만 적용.
