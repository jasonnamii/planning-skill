# Phase 4: 창의적 점프

논리적으로 수렴된 결론(P3 스파인) 위에 점프기술을 적용하여 차별화 요소를 만들거나, 복수 컨셉으로 분기한다.

---

## 핵심 원리

점프는 **수렴 위의 발산**이다. P3에서 "왜 이 방향인가"가 확정된 상태에서 "이 방향을 어떻게 더 날카롭게/독특하게/강력하게 만들 것인가"를 탐색한다. 수렴 없는 발산은 혼란, 발산 없는 수렴은 평범.

---

## 점프기술 메뉴

5개 기본 트리거 + 형 지정 옵션. trigger-dictionary 스킬을 통해 호출한다.

| 트리거 | 핵심 | 적합한 상황 | 예시 |
|--------|------|------------|------|
| **아날로지** | 다른 분야의 구조적 유사성 전이 | 새로운 관점 필요, 기존 프레임이 고착 | 물류 기획에 혈관 순환 구조 적용 |
| **맥가이버** | 제약 조건 내 자원 재조합 | 예산·시간·인력 제한, 기존 자원 활용 극대화 | 남는 공간+유휴 인력+기존 콘텐츠 재조합 |
| **모순기술** | 양립 불가능한 요구의 동시 해결 | "A도 하고 B도 해야 하는데 둘이 충돌" | 접근성↑과 프리미엄↑의 동시 달성 |
| **super-skill (자극설계)** | 인간 메커니즘 기반 설계 | 사용자 반응·행동 변화가 핵심인 기획 | 손실회피 활용한 이탈 방지 설계 |
| **넛지** | 선택 아키텍처 설계 | 행동 유도가 필요하지만 강제 불가 | 기본값 설정으로 참여율 향상 |

### 추천 로직

형이 점프기술을 직접 지정하지 않은 경우, P3 스파인과 갭 특성을 기반으로 1~2개를 추천한다.

```
P3 스파인: [확정된 스파인]
점프 추천:
① 맥가이버 — 이유: 예산 제약(P0)이 핵심 변수이므로 자원 재조합이 효과적
② 아날로지 — 이유: 유사 산업 벤치마크에서 구조 전이 가능성 높음

→ 적용할 기술을 선택해주세요. 또는 다른 기술을 지정해주세요.
```

---

## 실행 절차

### 1. 점프기술 선택

형 지정 또는 추천 승인.

### 2. 트리거 호출

trigger-dictionary 스킬에서 해당 트리거를 호출하여 프로토콜대로 실행.

### 3. 점프 결과 평가

점프기술 적용 결과를 P3 스파인과 대조:

| 평가 항목 | 질문 |
|-----------|------|
| 스파인 정합성 | 점프 결과가 스파인을 강화하는가, 이탈하는가? |
| 실현 가능성 | P0 제약 조건 내에서 실행 가능한가? |
| 차별화 효과 | 점프 전 대비 무엇이 달라졌는가? |

### 4. 복수 컨셉 분기 (선택)

점프 결과가 2개 이상의 방향을 열 때, 프레이밍 변수를 달리하여 2~4개 컨셉으로 분기.

**프레이밍 변수:**

| 변수 | 설명 |
|------|------|
| 수혜자 우선순위 | 누구를 먼저 만족시키는가 |
| 가치 축 | 어떤 가치를 최우선으로 두는가 |
| 제약 수용도 | 제약을 어디까지 수용/도전하는가 |
| 톤 | 보수적 vs 공격적 vs 실험적 |

각 컨셉은 상호 배타적이어야 한다 (A를 선택하면 B는 불가능한 수준의 차이).

### 5. 형 확인

점프 결과 또는 복수 컨셉을 형에게 제시. 승인 또는 조합 지시 후 P5로.

---

## 게이트: P4 → P5

| 조건 | 기준 |
|------|------|
| 통과 | 점프 결과 형 확인 완료 |
| 미충족 | 다른 점프기술 적용 또는 P3 결과로 복귀 |

---

## Gotchas

- **스파인 미확정 상태에서 점프:** 수렴 없이 발산하면 "영감은 많은데 방향이 없는" 결과. P3 게이트 통과가 전제.
- **점프기술 중복 적용:** 한 번에 3개 이상 적용하면 결과가 혼탁. 1~2개로 제한하고, 부족하면 형과 협의 후 추가.
- **복수 컨셉의 중복:** 컨셉 간 차이가 "표현만 다름" 수준이면 무의미. 프레이밍 변수 최소 2개가 달라야 한다.
- **점프 = 필수라는 착각:** 형이 P3에서 "마무리"를 선언하면 P4는 스킵. 모든 기획에 창의적 점프가 필요한 것은 아니다.
