# Phase 3: 논리적 수렴

리서치 결과를 하나의 스파인(핵심 메시지)으로 수렴하고, 기획안의 뼈대를 세운다. 이 Phase의 산출물이 기획안 v1.

---

## 핵심 원리

수렴 = "많은 정보를 요약"이 아니라 **"이 기획이 왜 이 방향이어야 하는가"의 답을 1문장으로 확정**하는 것. 스파인이 없으면 기획안은 "항목 나열"이 된다.

---

## 실행 절차

### 1. 스파인 후보 도출

P2의 팩트→결정→기획반영 테이블에서 "결정" 열을 종합하여 스파인 후보 2~3개를 추출한다.

**스파인 기준:**
- 기획 전체를 관통하는 1문장
- 리서치 근거에 의해 뒷받침됨
- 성공기준(P0)과 정합

**백본 트리거 호출:** trigger-dictionary에서 백본 트리거를 호출하여 스파인 후보의 구조적 건전성을 검증한다.

### 2. 형 승인

스파인 후보를 형에게 제시하고 방향을 확정받는다.

```
스파인 후보:
① [후보 1] — 근거: [P2 어떤 발견]
② [후보 2] — 근거: [P2 어떤 발견]
③ [후보 3] — 근거: [P2 어떤 발견]

추천: ①번 (이유: ...)
```

형이 선택하거나 수정 방향을 제시하면 확정.

### 3. 기획안 뼈대 (v1) 작성

확정된 스파인을 기반으로 기획안 골격을 세운다.

**스켈레톤 트리거 호출:** trigger-dictionary에서 스켈레톤 트리거를 호출하여 구조를 설계한다.

v1 구성:
- 스파인 (1문장)
- 진단 (왜 이 기획이 필요한가 — P2 리서치 핵심)
- 전략 방향 (스파인에서 파생된 큰 방향 2~4개)
- 각 방향별 핵심 액션 (제목 수준, 내용은 P5에서)
- 실행 조건 / 제약 (P0에서 가져옴)

### 4. 분기 판단

| 상황 | 경로 |
|------|------|
| 형이 "여기서 마무리" 또는 논리적 수렴으로 충분 | → P5 직행 |
| 차별화·창의적 요소 필요 | → P4 점프 |
| 스파인이 약함 (리서치 부족) | → P2 추가 리서치 |

---

## 게이트: P3 → P4 또는 P5

| 전환 | 조건 |
|------|------|
| P3→P4 | 스파인 확정 + 형 승인 + 창의적 점프 필요 |
| P3→P5 | 스파인 확정 + 형 "마무리" 또는 도메인 특성상 수렴만으로 충분 |
| P3→P2 (역행) | 스파인 후보 전부 약함 → 리서치 보강 |

---

## Gotchas

- **스파인 없이 구조부터:** "1장 현황, 2장 전략, 3장 실행..."은 스파인이 아니라 목차. 목차는 스파인에서 파생되어야 한다.
- **리서치 결과 나열 = 수렴 아님:** P2의 팩트를 순서대로 옮기는 것은 수렴이 아니다. "이 팩트들이 종합적으로 의미하는 바"를 추출해야 한다.
- **형 승인 없이 진행:** 스파인은 기획의 방향을 결정하므로 반드시 형의 확인을 거친다. 자율 진행하면 P4~P5 전체가 방향 오류.
