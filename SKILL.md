# Planning Skill

범용 기획 파이프라인. 형의 기획 사고 흐름을 6 Phase로 프로토콜화한 오케스트레이터.

이 스킬의 핵심 역할: **기획의 "언제·무엇을" 판단**하고, **"어떻게"는 전문 스킬/트리거에 위임**한다.

## 게이트 (Phase 간 통과 조건)

게이트를 두는 이유: 리서치 없이 기획하거나, 스파인 없이 축 나열하는 실패를 구조적으로 차단한다.

전환통과 조건미충족 시게이트 유형P0→P1배경·의뢰자·제약·성공기준 중 3/4 확보부족 항목 질문HardP1→P2갭 매트릭스 완성 + 🔴 1개 이상갭 없으면 "기획 불필요" 보고**Hard**P2→P3축별 최소 1건 교차 확인 + 확신도 50↑추가 리서치 축 제안**Soft**P3→P4스파인 확정 + 형 방향 승인스파인 후보 2\~3개 → 형 선택SoftP3→P5형 "여기서 마무리" 또는 수렴 완료 판단P4 스킵, P5 직행SoftP4→P5점프 결과 형 확인다른 점프기술 적용 또는 P3 복귀Soft

**게이트 정책:**

- **Hard gate (P1→P2):** 필수 통과. 미충족 시 다음 Phase 진입 불가.
- **Soft gate (P2→P3, P3↑):** 경고 제시 후 진행 가능. 형이 판단하여 스킵 가능.

**P2 리서치 반려 시 복귀 경로**:P2 리서치 결과가 불충분하면:

1. **P1의 갭 프레임만 수정**
2. **P1 전체 재실행 아님**
3. **수정된 갭으로 P2 재진입** (축별 조사 재수행)

## PRE_WRITE — P5 기획안 작성 직전 룰

**목적:** 기획안 v1·v2·v3 *작성 시점*에 단문·단언형·수치+출처 강제. SHE·제출청소 트리거의 사후정리 + PRE_WRITE 사전생성 = 이중 게이트.

### 5룰 (P5 작성 직전 강제)

| # | 룰 | 기획 결로 변환 | FAIL 신호 |
|---|---|---|---|
| 1 | **스파인 단문** | 핵심 서사 1줄 ≤25자. 단언형 | "추진하고자 한다" 완곡 / 30자+ |
| 2 | **소제목 단언형** | 모든 소제목 ≤20자, 결론·주장형 | 라벨식·질문식 |
| 3 | **수치+출처 강제** | 모든 주장 = 수치 + 1차 출처 1조 | "큰 효과" 추상 |
| 4 | **문장당 1명제** | 본문 1행 = 1주장 | AND·OR 묶기 |
| 5 | **AI식 사전회피** | "전사적 차원에서", "유연하게 대응", "지속 가능한" — 작성 단계 차단 | 컨설팅 관용어 |

### Phase별 PRE_WRITE

- **P3 수렴**: 스파인 1줄 ≤25자 강제. 형 승인 게이트 통과 조건
- **P5 진화**: v1·v2·v3 모두 5룰. 진화 = PRE_WRITE 통과 카피의 압축
- **제출청소 트리거**: 5룰 위반 grep = 청소 1차 필터

### 자체검증

| # | 체크 | 위반 |
|---|------|------|
| 1 | 스파인 ≤25자, 단언형? | 초과·완곡 = 재수렴 |
| 2 | 소제목 단언형? | 라벨식 = 재작성 |
| 3 | 수치+출처? | 누락 = 자료 보완 |
| 4 | AI식 어휘 hit? | hit ≥ 1 = 평문 변환 |

## Gotchas

- **리서치 없이 기획 착수:** P1 갭 매트릭스만으로 바로 기획안 작성 시도. 반드시 P2를 거쳐야 "왜 이 방향인가"의 근거가 생긴다.
- **스파인 미확정 상태로 P4 진입:** 점프기술은 수렴된 결론 위에 적용해야 효과. 스파인 없이 점프하면 발산만 됨.
- **P4를 매번 강제:** 형이 "여기서 마무리"라고 하면 P3→P5 직행. 점프가 항상 필요한 건 아니다.
- **포맷 임의 결정:** .md 외 포맷이나 스타일 체계는 반드시 형이 지정. 스킬이 "HTML div 적용하겠습니다" 같은 자의적 판단 금지.
- **도메인 라우팅 무시:** 정책기획 신호가 명확한데 플래닝 스킬로 강행하면 ABCD축·제출청소 등 도메인 프로토콜 누락.
- **P0 과잉 세팅:** 프로젝트 CLAUDE.md가 이미 있으면 P0 대부분 스킵 가능. 매번 처음부터 좌표계를 세우지 않는다.
- **QC 결과를 산출물에 포함:** P5의 기획언어 금지패턴 체크, 수치 정합성 검증 등은 내부 검증 절차다. 산출물에 "금지패턴 준수 확인" 테이블이나 "사용 안 함" 목록을 넣으면 금지 표현이 산출물에 노출된다. QC는 작성 과정에서 수행하고, 그 결과(체크리스트·검증 로그)는 산출물에서 제거한다.
- **라우팅 위임 시 대상 스킬 내부 구조 인용:** 위임을 제안할 때 대상 스킬의 Phase 번호, 단계명, 내부 프로토콜명 등을 산출물에 인용하지 않는다. "정책기획 전문 파이프라인이 더 적합할 수 있다" 수준으로 안내하고, 형이 상세를 원하면 해당 스킬 호출로 대응한다.
